# Data Preparation V3

This is the third iteration of the data pipeline for the cube and cylinder detection model. V3 upgrades the extraction step to **combine all existing data sources** into a single unified dataset, instead of processing one session at a time.

> **Fresh machine?** See [`SETUP.md`](../../SETUP.md) at the project root for first-time setup and virtual environment activation.

---

## What's New in V3

| | V2 | V3 |
|---|---|---|
| Script | `1_extract_frames.py` | `1_extract_and_combine.py` |
| Sources | Single recording session | New recordings + V1 videos + V2 images |
| Frame skip | Configurable (`FRAME_SKIP`) | Always extracts every frame |
| Arc class | Included | Dropped (cubes + cylinders only) |

The final `data.yaml` is what gets passed to `train.py`.

## Step 1 — Extract & Combine (`1_extract_and_combine.py`)

Pulls frames from three sources and writes them into one combined pool per class.

```powershell
python "Object Detection Pipeline\Data_Preparation_V3\scripts\1_extract_and_combine.py"
```

**What it does — in three phases:**

| Phase | Source | Action |
|---|---|---|
| 1 | `recordings/cubes`, `recordings/cylinders` | Extract every frame from new recordings |
| 2 | `Data_Preparation_V1/data/raw_videos/Cube`, `.../Cylinder` | Extract every frame from V1 footage (Arc ignored) |
| 3 | `Data_Preparation_V2/data/images/cubes`, `.../cylinders` | Copy existing V2 JPEG images directly |

**Output:** `Data_Preparation_V3/data/images/{cubes,cylinders}/`

Images are named with their source embedded in the filename (e.g. `cubes_src_recording1_00001.jpg`, `cubes_v2src_00142.jpg`) so you can always trace where a frame came from.

---

## Step 2 — Split Dataset (`2_split_dataset.py`)

Shuffles and splits the combined images into train / val / test.

```powershell
python "Object Detection Pipeline\Data_Preparation_V3\scripts\2_split_dataset.py"
```

- Split ratio: **75% train / 15% val / 15% test**
- Uses a fixed random seed — fully reproducible
- Copies files (originals in `data/images/{class}/` stay intact)
- Output: `data/images/train|val|test/{class}/`

---

## Step 3 — Auto-Annotate (`3_auto_annotation.py`)

Runs **GroundingDINO** to generate YOLO-format bounding box labels for every image.

```powershell
python "Object Detection Pipeline\Data_Preparation_V3\scripts\3_auto_annotation.py"
```

GroundingDINO is a zero-shot open-vocabulary detection model — it finds objects using only text prompts, without needing to be pre-trained on your specific classes. The script queries it with `"cube"` and `"cylinder"` and writes one `.txt` label file per image in YOLO format (class index + normalised `cx cy w h`).

**Key config** (edit at top of script):

| Variable | Default | Description |
|---|---|---|
| `BOX_THRESHOLD` | `0.35` | Minimum detection confidence |
| `TEXT_THRESHOLD` | `0.25` | Minimum text-match confidence |
| `PROMPT_MAP` | `{cubes: "cube", cylinders: "cylinder"}` | Text prompts sent to DINO |
| `CLASS_ORDER` | `["cubes", "cylinders"]` | Sets class IDs: cube=0, cylinder=1 |

**Output:** `data/dataset/` — a complete YOLO-ready folder with `data.yaml` included.

## Checking Dataset State

Run `dataset_stats.py` at any point to see the current state of the dataset:

```powershell
python "Object Detection Pipeline\dataset_stats.py"
```

Reports:
- Raw video counts per class
- Image counts per class × split (train / val / test)
- Label coverage (labels generated vs images)
- Pipeline readiness checklist

> **Note:** The `DATA_DIR` path at the top of `dataset_stats.py` must point to your local `Data_Preparation_V3/data` folder.

## Troubleshooting

| Error | Cause | Fix |
|---|---|---|
| `running scripts is disabled` | PowerShell policy | `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` |
| `python not found` | Python not in PATH | Use `py -3.11` instead of `python` |
| `Could not open <video>` | Corrupt or missing video file | Check file exists and is a valid `.mp4`/`.avi` |
| GroundingDINO import error | C++ build failed | Install `groundingdino-py` (pure-Python wheel) before other deps |
| `torch not found` | PyTorch not installed | Install via `--index-url` as shown in Step 0 |
| `CUDA out of memory` | GPU VRAM saturated | Close other GPU apps or reduce batch size in annotation script |
| `UnicodeEncodeError` | Windows console encoding | Already handled in all V3 scripts |
