# PDE3802 — Pose Estimation Project: Model V2 Data Pipeline

A complete guide to preparing the **Model V2** training dataset for cube and cylinder pose estimation — from raw recordings through to annotated YOLO-ready data.

> **Fresh machine?** See [`SETUP.md`](../SETUP.md) at the project root for the full first-time setup and every-session activation steps.

---

## Data Preparation V2 Pipeline

All scripts live in `Data_Preparation_V2/scripts/`. Run them **in order** from the **project root**.

### Folder Structure

```
Data_Preparation_V2/
├── scripts/
│   ├── 1_extract_frames.py      ← Step 1: extract frames from raw videos
│   ├── 2_split_dataset.py       ← Step 2: split frames 75/15/15
│   ├── 3_auto_annotation.py     ← Step 3: auto-annotate with GroundingDINO
│   └── 3_dataset_stats.py       ← Utility: check dataset state at any time
├── data/
│   ├── raw_videos/
│   │   ├── cubes/               ← raw .mp4 recordings
│   │   └── cylinders/           ← raw .mp4 recordings
│   ├── images/
│   │   ├── cubes/               ← flat extracted frames (before split)
│   │   ├── cylinders/
│   │   ├── train/{cubes,cylinders}/
│   │   ├── val/{cubes,cylinders}/
│   │   └── test/{cubes,cylinders}/
│   ├── labels/                  ← generated by 3_auto_annotation.py
│   └── dataset/                 ← final YOLO-ready merged dataset
│       ├── train/{images,labels}/
│       ├── val/{images,labels}/
│       ├── test/{images,labels}/

├── requirements_dataprep.txt    ← lightweight deps (opencv, tqdm)
└── requirements_annotation.txt  ← full ML deps (torch, autodistill, DINO)
```

## Step 0 — Install Dependencies

```powershell
pip install -r Data_Preparation_V2\requirements_dataprep.txt
```

### Annotation (Step 3) — install in this order
```powershell

pip install torch==2.3.1 torchvision==0.18.1 --index-url https://download.pytorch.org/whl/cu121

# 2. GroundingDINO pure-Python wheel (avoids C++ build errors on Windows)
pip install groundingdino-py

# 3. Everything else
pip install -r Data_Preparation_V2\requirements_annotation.txt
```

## Step 1 — Extract Frames (`1_extract_frames.py`)

Reads every video in `data/raw_videos/{cubes,cylinders}/` and saves frames as JPEGs into `data/images/{class}/`.

```powershell
python Data_Preparation_V2\scripts\1_extract_frames.py
```

**Key config** (edit at the top of the script):

| Variable | Default | Description |
|---|---|---|
| `FRAME_SKIP` | `3` | Extract every Nth frame — `3` gives ~10 fps from a 30 fps source |
| `JPEG_QUALITY` | `95` | Output image quality (0–100) |

**Output naming:** `cubes_0001.jpg`, `cubes_0002.jpg`, … (sequential across all videos per class)

---

## Step 2 — Split Dataset (`2_split_dataset.py`)

Shuffles and splits the flat frames into `train / val / test` at **75 / 15 / 15** per class.

```powershell
python Data_Preparation_V2\scripts\2_split_dataset.py
```

- Uses a fixed random seed (`42`) — fully reproducible
- Copies files (originals in `data/images/{class}/` remain intact)
- Output goes into `data/images/train|val|test/{class}/`

---

## Step 3 — Auto-Annotate (`3_auto_annotation.py`)

Runs **GroundingDINO** via `autodistill` to generate YOLO bounding-box labels, then assembles the final `data/dataset/` folder.

```powershell
python Data_Preparation_V2\scripts\3_auto_annotation.py
```

**First run downloads ~700 MB of model weights** — this only happens once.

**Key config** (edit at the top of the script):

| Variable | Default | Description |
|---|---|---|
| `BOX_THRESHOLD` | `0.35` | Min confidence for a detection box |
| `TEXT_THRESHOLD` | `0.25` | Min confidence for text-to-box match |
| `PROMPT_MAP` | `{cubes: "cube", cylinders: "cylinder"}` | Text prompts for DINO |
| `CLASS_ORDER` | `["cubes", "cylinders"]` | Sets class IDs: cubes=0, cylinders=1 |

**Output:** `data/dataset/` with `data.yaml` ready for YOLO training.

## Troubleshooting

| Error | Cause | Fix |
|---|---|---|
| `running scripts is disabled` | PowerShell policy | `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` |
| `python not found` | Python not in PATH | Use `py -3.11` instead of `python` |
| `pip not found` | venv not activated | Run `.\venv\Scripts\Activate.ps1` from the project root |
| `Activate.ps1` not found | Wrong directory | `cd` to the project root, not inside the `venv` folder |
| `UnicodeEncodeError` | Windows console encoding | Already fixed in all V2 scripts |
| GroundingDINO import error | C++ build failed | Install `groundingdino-py` (pure-Python wheel) first |
| `torch not found` | PyTorch not installed | Install PyTorch via `--index-url` before other annotation deps |
| `CUDA out of memory` | GPU VRAM too low | Close other GPU apps or use CPU-only PyTorch |
