# Train — YOLO Model Training

This folder contains the unified training script for all YOLO models used in the object detection pipeline. Three models are trained and compared: **YOLOv8n**, **YOLO26n**, and **YOLOv5n**.

> **Before training**, make sure you have a completed and annotated dataset from `Data_Preparation_V3`. The `data.yaml` in this folder must point to the correct dataset path on your machine.

---

## Files

```
Train/
├── README.md        ← this file
├── train.py         ← unified training script for all 3 models
└── data.yaml        ← dataset config (update path before training)
```

---

## Prerequisites

### 1. Update `data.yaml`

Open `data.yaml` and set `path` to the absolute path of your `Data_Preparation_V3/data` folder on this machine:

```yaml
path: C:/Users/<you>/path/to/Data_Preparation_V3/data
train: images/train
val:   images/val
test:  images/test
nc: 2
names: ['cube', 'cylinder']
```

### 2. Update `train.py` config block

At the top of `train.py`, update `DATA_YAML` and `GIT_EXE` for this machine:

```python
DATA_YAML = r"C:\path\to\Train\data.yaml"
GIT_EXE   = r"C:\path\to\Git\cmd\git.exe"
DEVICE    = 0        # 0 = first GPU, 'cpu' = CPU only
BATCH     = 32       # reduce if you get CUDA OOM errors
```

### 3. Install dependencies

```powershell
# Ultralytics (YOLOv8 + YOLO26)
pip install ultralytics --upgrade

# ONNX export support
pip install onnx onnxruntime

# YOLOv5 is auto-cloned by the script — no manual install needed
```

## CLI Commands

### Training

```powershell
# Train all three models sequentially (v8 → v26 → v5)
python Train\train.py

# Train a single model
python Train\train.py --train v8
python Train\train.py --train v26
python Train\train.py --train v5

# Train all models (explicit)
python Train\train.py --train all
```

### Export to ONNX

```powershell
# Export all trained models to ONNX (run after training)
python Train\train.py --train none --export

# Train + export in one go
python Train\train.py --train all --export
```

### Compare Results

```powershell
# Print a side-by-side mAP comparison table for all trained models
python Train\train.py --train none --compare

# Full pipeline: train, export, then compare
python Train\train.py --train all --export --compare
```

## What Gets Trained

| Model | Architecture | Deployment Target | Blob Support |
|---|---|---|---|
| YOLOv8n | Ultralytics v8, anchor-free | OAK-D Lite (blob) | ✅ Yes |
| YOLO26n | YOLO v2.6, fast CPU inference | Laptop CPU / Raspberry Pi (ONNX) | ❌ Not yet |
| YOLOv5n | Classic anchor-based, best OAK-D docs | OAK-D Lite (blob) | ✅ Yes |

All models are trained with the same settings for a fair comparison:

| Parameter | Value |
|---|---|
| Epochs | 100 |
| Early stopping patience | 20 |
| Image size | 640 × 640 |
| Batch size | 32 (RTX 5080) — reduce on lower VRAM |
| Device | GPU 0 (or `'cpu'`) |

---

## Augmentation

All three models use the same augmentation settings (defined once in `train.py` as the `AUG` dict):

| Augmentation | Value | Purpose |
|---|---|---|
| `degrees` | 45 | Rotation ±45° — objects recorded at many angles |
| `translate` | 0.2 | Shift up to ±20% of frame |
| `scale` | 0.5 | Zoom in/out ±50% — accounts for 30/60/90cm recording distances |
| `fliplr` | 0.5 | Horizontal flip 50% of the time |
| `flipud` | 0.1 | Vertical flip 10% (less common viewpoint) |
| `perspective` | 0.001 | Subtle perspective warp |
| `hsv_h/s/v` | 0.1 / 0.4 / 0.3 | Colour & brightness jitter for lighting variation |
| `mosaic` | 1.0 | Mosaic always on — tiles 4 images together |
| `mixup` | 0.05 | Blends two images with soft labels (5% of batches) |
| `erasing` | 0.1 | Random region erasure (10%) |

For YOLOv5, these are written to `hyp_v5.yaml` automatically since v5 reads hyperparameters from a YAML file.

## Deploying on OAK-D Lite

After exporting to ONNX, convert to a `.blob` file for the OAK-D using the Luxonis online converter:

1. Go to [blobconverter.luxonis.com](https://blobconverter.luxonis.com/)
2. Upload `best.onnx` (from yolov8n or yolov5n)
3. Settings: **Shaves = 6**, **FP16**, **RVC2**
4. Download the `.blob` and place it in your inference script folder

> YOLO26n blob conversion is **not yet supported** by the Luxonis compiler. Use YOLO26n ONNX for CPU/Raspberry Pi inference only.

## Troubleshooting

| Error | Cause | Fix |
|---|---|---|
| `CUDA out of memory` | Batch too large for GPU | Reduce `BATCH` to 16 or 8 in `train.py` |
| `yolov5 git clone failed` | Git not found | Set `GIT_EXE` to the full path to `git.exe` |
| `data.yaml not found` | Wrong `DATA_YAML` path | Update path at the top of `train.py` |
| `UnicodeDecodeError` on Windows | Console encoding | Already handled — script forces UTF-8 |
| `yolo26n.pt not found` | Model not downloaded | Run `python -c "from ultralytics import YOLO; YOLO('yolo26n.pt')"` first |
| ONNX export fails for v5 | Wrong opset | Script uses opset 12 — confirm `onnx>=1.12` is installed |
